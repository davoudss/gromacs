00202    elamstatsMINVAR = using Barker + minimum variance for weights

00214 /* How moves in lambda are calculated:
00215    elmovemcMETROPOLIS - using the Metropolis criteria, and 50% up and down
00216    elmovemcBARKER - using the Barker criteria, and 50% up and down
00217    elmovemcGIBBS - computing the transition using the marginalized probabilities of the lambdas

00372 /* Different formats for table data. Cubic spline tables are typically stored
00373  * with the four Y,F,G,H intermediate values (check tables.c for format), which
00374  * makes it easy to load with a single 4-way SIMD instruction too.
00375  * Linear tables only need one value per table point, or two if both V and F
00376  * are calculated. However, with SIMD instructions this makes the loads unaligned,
00377  * and in that case we store the data as F, D=F(i+1)-F(i), V, and then a blank value,
00378  * which again makes it possible to load as a single instruction.
00379  */
