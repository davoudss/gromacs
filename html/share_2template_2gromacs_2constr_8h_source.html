00153 /*
00154  * When econq=econqCoord constrains coordinates xprime using th
00155  * directions in x, min_proj is not used.
00156  *
00157  * When econq=econqDeriv, calculates the components xprime in
00158  * the constraint directions and subtracts these components from min_proj.
00159  * So when min_proj=xprime, the constraint components are projected out.
00160  *
00161  * When econq=econqDeriv_FlexCon, the same is done as with econqDeriv,
00162  * but only the components of the flexible constraints are stored.
00163  *
00164  * When bMolPBC=TRUE, assume that molecules might be broken: correct PBC.
00165  *
00166  * delta_step is used for determining the constraint reference lengths
00167  * when lenA != lenB or will the pull code with a pulling rate.
00168  * step + delta_step is the step at which the final configuration
00169  * is meant to be; for update delta_step = 1.
00170  *
00171  * If v!=NULL also constrain v by adding the constraint corrections / dt.
00172  *
00173  * If vir!=NULL calculate the constraint virial.
00174  *
00175  * if veta != NULL, constraints are done assuming isotropic pressure control
00176  * (i.e. constraining rdot.r = (v + veta*r).r = 0 instead of v
00177  *
00178  * Init_constraints must have be called once, before calling constrain.
00179  *
00180  * Return TRUE if OK, FALSE in case of shake error
00181  *
00182  */
