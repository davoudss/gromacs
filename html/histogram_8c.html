void gmx_histogram_resample_dblbw ( gmx_histogram_t **  destp,
gmx_histogram_t *  src,
gmx_bool  bIntegerBins 
)
Parameters:
[out]destpDestination histogram.
[in]srcSource histogram.
[in]bIntegerBinsControl bin center position.

Uses src to create a new histogram that has double the binwidth. If bIntegerBins is TRUE, the first new bin will be centered at src->start, otherwise the left edge will be at src->start.

This function is mostly useful if one needs to sample both the histogram and the cumulative histogram at same points. To achieve this, first sample a histogram with half the desired binwidth, and then use this function to create two different verions of it with different values of bIntegerBins. gmx_histogram_write_array() and gmx_histogram_write_cum_array() can now used to write out the values correctly at identical values.

Should be called only after gmx_histogram_finish() has been called. src should have been sampled without gmx_histogram_set_integerbins().

void gmx_histogram_write_array ( FILE *  fp,
int  n,
gmx_histogram_t *  h[],
gmx_bool  bValue,
gmx_bool  bErrors 
)
Parameters:
[in]fpOutput file.
[in]nNumber of histograms in h.
[in]hArray of histograms to write.
[in]bValueIf TRUE, histogram values are written.
[in]bErrorsIf TRUE, histogram errors are written.

All the histograms in the array h should have the same bin widths and left edges, otherwise the behavior is undefined. The output format is one bin per line. The first number on the line is the center of the bin, followed by one or two numbers for each histogram (depending on bValue and bErrors). If both bValue and bErrors are both TRUE, the values are written before the errors.
