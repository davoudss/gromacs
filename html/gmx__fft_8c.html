Transpose 2d multi-element matrix.

This routine is very similar to gmx_fft_transpose_2d(), but it supports matrices with more than one data value for each position. It is extremely useful when transposing the x/y dimensions of a 3d matrix - in that case you just set nelem to nz, and the routine will do and x/y transpose where it moves entire columns of z data

This routines works when the matrix is non-square, i.e. nx!=ny too, without allocating an entire matrix of work memory, which is important for huge FFT grid.

For performance reasons you need to provide a small workarray with length at least 2*nelem (note that the type is char, not t_complex).

Parameters:
in_dataInput data, to be transposed
out_dataOutput, transposed data. If this is identical to in_data, an in-place transpose is performed.
nxNumber of rows before transpose
nyNumber of columns before transpose
nelemNumber of t_complex values in each position. If this is 1 it is faster to use gmx_fft_transpose_2d() directly.
workWork array of length 2*nelem, type t_complex.
Returns:
GMX_SUCCESS, or an error code from gmx_errno.h
