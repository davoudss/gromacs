Detailed Description

Partial implementation of MPI using only threads.

See the MPI specification at http://www.mpi-forum.org/docs/docs.html for an explanation of what these functions do.

Because this is a thread-based library, be very careful with global variables and static variables in functions: they will be shared across all threads and lead to conflicts if not properly mutex-ed or barrier-ed out.

See also:
http://www.mpi-forum.org/docs/docs.html for MPI documentation.

enum tMPI_Affinity_strategy

tMPI initialization thread affinity strategy.

Used in the tMPI_Init_affinity() and tMPI_Init_fn_affinity() functions, to control how affinity is set. The default tMPI_Init() and tMPI_Init_fn() functions use the TMPI_AFFINITY_ALL_CORES strategy.

These strategies are fairly basic. For more flexibility, use the tMPI_Set_affinity() function.

Enumerator:
TMPI_AFFINITY_NONE 

Do not set any thread affinity

TMPI_AFFINITY_ALL_CORES 

Only set affinity if the number of threads is equal to the number of hardware threads (cores + hyperthreads). This is the only safe way to set thread affinity, without clashes between multiple instances of the same program.

TMPI_AFFINITY_NONE

TMPI_EXPORT int tMPI_Init ( int *  argc,
char ***  argv,
int(*)(int, char **)  start_function 
)

Traditional MPI initializer; spawns threads that start at the given function.

Seeks the argument '-nt n', where n is the number of threads that will be created. If n==0, the number of threads will be the recommended number of threads for this platform as obtained from tMPI_Get_recommended_ntreads().

The new threads then run the function start_function, with the original argc and argv. This function could be main(), or any other function; calling this function again - whether from the started threads or from the main thread - has no effect.

On platforms that support thread affinity setting, this function will use the 'all-cores' affinity strategy: it will only set thread affinity if the number of threads is equal to the number of hardware threads (cores + hyperthreads).

Parameters:
[in]argcargc of original main() invocation, or NULL
[in]argvargv of original main() invocation, or NULL.
[in]start_functionStarting function of type int start_function(int argc, char *argv[]);
Returns:
TMPI_SUCCESS on success, TMPI_FAILURE on failure.
TMPI_EXPORT int tMPI_Init_fn ( int  main_thread_returns,
int  N,
tMPI_Affinity_strategy  aff_strategy,
void(*)(void *)  start_function,
void *  arg 
)

Generic init function thread MPI intializer and thread spawner.

Creates N threads (including main thread) that run the function start_function, which takes a void* argument, given by arg. The function start_function also gets called by the main thread. When the function start_function returns it, will behave as if tMPI_Finalize is called, and if it's a sub-thread it will stop running.

If N==0, the number of threads will be the recommended number of threads for this platform as obtained from tMPI_Get_recommended_ntreads().

Note that thread affinity strategy only has an effect when this is supported by the underlying platform. As of yet (2012), this is not the case for Mac OS X, for example.

Parameters:
[in]main_thread_returnswhether the control in the main thread should return immediately (if true), or the start_function() should be called from the main thread, too (if false).
[in]NThe number of threads to start (or 0 to automatically determine this).
[in]aff_strategyThe thread affinity strategy to use.
[in]start_functionThe function to start threads at (including main thread if main_thread_returns).
[in]argAn optional argument for start_function().
Returns:
TMPI_FAILURE on failure, TMPI_SUCCESS on succes (after all threads have finished if main_thread_returns=true).
