tMPI_Atomic_t once

Detailed Description

One-time initialization data for thread.

This is an opaque datatype which is necessary for tMPI_Thread_once(), but since it needs to be initialized statically it must be defined in the header. You will be sorry if you touch the contents. Variables of this type should always be initialized statically to TMPI_THREAD_ONCE_INIT.

This type is used as control data for single-time initialization. The most common example is a mutex at file scope used when calling a non-threadsafe function, e.g. the FFTW initialization routines.
