00038 /* This is the innermost loop contents for the 4 x N atom SIMD kernel.
00039  * This flavor of the kernel duplicates the data for N j-particles in
00040  * 2xN wide SIMD registers to do operate on 2 i-particles at once.
00041  * This leads to 4/2=2 sets of most instructions. Therefore we call
00042  * this kernel 2x(N+N) = 2xnn
00043  *
00044  * This 2xnn kernel is basically the 4xn equivalent with half the registers
00045  * and instructions removed.
00046  *
00047  * An alternative would be to load to different cluster of N j-particles
00048  * into SIMD registers, giving a 4x(N+N) kernel. This doubles the amount
00049  * of instructions, which could lead to better scheduling. But we actually
00050  * observed worse scheduling for the AVX-256 4x8 normal analytical PME
00051  * kernel, which has a lower pair throughput than 2x(4+4) with gcc 4.7.
00052  * It could be worth trying this option, but it takes some more effort.
00053  * This 2xnn kernel is basically the 4xn equivalent with
00054  */
