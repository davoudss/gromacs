00046 /*
00047  * Sort_bonds sorts all bonded force parameters in order of ascending
00048  * atom id of the maximum atom id specified in a bond per type bond.
00049  *
00050  * If, for example, for a specific bond type the following bonds are specified:
00051  *    bond1 between atoms 15, 16, 12, 18 and 20
00052  *    bond2 between atoms 14, 13, 12, 18 and 19
00053  *    bond3 between atoms 17, 11, 19, 21 and 15
00054  * then the maximum atom id for each bond would be:
00055  *    bond1: 20
00056  *    bond2: 19
00057  *    bond3: 21
00058  * so order in which the bonds will be sorted is bond2, bond1, bond3
00059  *
00060  * This routine is used to determine to which node a bonds should be
00061  * allocated. For the distribution of bonds it is necessary to keep all
00062  * the needed atoms when calculating a bonded force on one node. In
00063  * this way we prevent communication overhead in bonded force calculations.
00064  *
00065  */
