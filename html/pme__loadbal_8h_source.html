00050 /* Try to adjust the PME grid and Coulomb cut-off.
00051  * The adjustment is done to generate a different non-bonded PP and PME load.
00052  * With separate PME nodes (PP and PME on different processes) or with
00053  * a GPU (PP on GPU, PME on CPU), PP and PME run on different resources
00054  * and changing the load will affect the load balance and performance.
00055  * The total time for a set of integration steps is monitored and a range
00056  * of grid/cut-off setups is scanned. After calling pme_load_balance many
00057  * times and acquiring enough statistics, the best performing setup is chosen.
00058  * Here we try to take into account fluctuations and changes due to external
00059  * factors as well as DD load balancing.
00060  * Returns TRUE the load balancing continues, FALSE is the balancing is done.
00061  */
