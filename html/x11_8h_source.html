00094 t_x11 *GetX11(int *argc, char *argv[]);
00095 /* x11 is a struct / function-set that manages a number of windows.
00096  * more or (presumably) less like Xt does, but since x11 uses only
00097  * Xlib calls, it is *PORTABLE* software.
00098  *
00099  * The x11 struct is in principle Object Oriented, in that the functions
00100  * are member of the struct. This makes the software a little more
00101  * managable. Because of portability I decided not to use C++, even
00102  * though it would be much nicer to work with in the X-Bizz.
00103  *
00104  * Here's the description of how to use the x11 struct
00105  * 1. Call the GetX11 routine, with the argc and argv from your main.
00106  *    This will sort out the X-arguments on the command line and remove
00107  *    them from the command line. When the routine returns, only the
00108  *    application specific arguments should be left. Thi opens the
00109  *    display, selects a font, creates a Graphics Context and also sets
00110  *    the colours listed above in the global variables.
00111  * 2. Call x11->RegisterCallback for each window you want to have
00112  *    managed by x11. You have to create a Callback routine for your
00113  *    application that handles *ONE* event at a time. The idea is that
00114  *    each window has it's own Callback which is not polluted by code
00115  *    for other windows, but it is of course entirely possible to have
00116  *    one Callback routine for a number of windows (eg. when you need
00117  *    to know something about your children).
00118  * 3. Call x11->SetInputMask. This comes in place of the normal
00119  *    XSelectInput, because it enables x11 to manually decide which
00120  *    events are passed to the windows. With the x11->GetInputMask,
00121  *    x11->SetInputMask combination, a child window can temporarily
00122  *    disable mouse and keyboard input for it's parent, while allowing
00123  *    redraw events to pass through for instance. Hereby a simple way
00124  *    for creating application modal child windows is implemented.
00125  * 4. Call x11->MainLoop. This will call every callback function as
00126  *    appropriate. When a window receives a message, that makes it decide
00127  *    to terminate it should call x11->UnRegisterCallback, in order to
00128  *    tell the x11 Manager that it does not want to receive any more
00129  *    events. It is up to the window to destroy itself. The MainLoop
00130  *    routine exits when there are no more windows to manage, i.e. when
00131  *    all routines have called UnRegisterCallback, OR when one Callback
00132  *    routine returns non-zero (TRUE).
00133  * 5. Call x11->CleanUp. This closes the display, and frees all
00134  *    memory allocated by x11 before.
00135  */
