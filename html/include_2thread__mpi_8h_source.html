00038 /*
00039    thread_mpi is a cross-platform threading library for applications in
00040    high-performance computing. It supports:
00041 
00042    - Cross-platform thread primitives (thread creation, mutexes, spinlocks,
00043      barriers, thread-local storage, etc.).
00044    - Cross-platform atomic operations (compare-and-swap, add-return, etc) for
00045      safe lock-free synchronization.
00046    - An implementation of (currently, much of) MPI, either as a drop-in
00047      replacement, or for use in conjunction with a networked MPI
00048      implementation.
00049    - Shared-memory allocation and memory management (planned, as of now).
00050    - Basic lock-free data structures (planned, as of now).
00051 
00052    Because it can be used as a drop-in replacement for MPI, existing codes
00053    using MPI can start using thread_mpi without major changes in the
00054    source code, assuming -- and this is a big assumption -- that the code
00055    is thread-safe.
00056 
00057    Alternatively, networked MPI calls can be used in conjunction with
00058    thread_mpi calls (simply by using
00059     "#include <thread_mpi.h>"
00060    instead of
00061     "#include <tmpi.h>"
00062    and pre-fixing all thread_mpi MPI-like calls with tMPI instead of MPI.
00063 
00064    The availability of both MPI calls and shared-memory constructs makes it
00065    possible to transition (relatively) seamlessly from an MPI-style code
00066    to code that's optimal on multicore CPUs.
00067 
00068    Although MPI-style message passing isn't neccesarily optimal for
00069    performance on shared-memory systems, the MPI communicator concept and
00070    its emphasis on collective operations makes sense even when computing on
00071    one machine with multiple cores. The communicator forms the basis for
00072    the shared-memory allocation and lock-free data structure implementations
00073    in thread_mpi.
00074 
00075    Although usable as a stand-alone library, thread_mpi is designed to
00076    be incorporated in the code tree, eliminating any external build
00077    requirements. The BSD-style license that this library is distributed
00078    with reflects this.
00079 
00080    The atomic operations (such as compare-and-swap) are supported on:
00081    - gcc on x86, x86_64, PowerPC and Itanium.
00082    - Intel compilers on x86, x86_64 and Itanium.
00083    - xlc on PowerPC.
00084    - (partial) HP/UX compilers on Itanium.
00085  */
